# Phase 1 Data Model — Reproducible Install & Unified Config

The system has exactly **one authoritative entity** in 001: the **config/wiring entity**,
stored in `platform.yaml`. Everything else here is either a sub-structure of it or a
**derived** (non-authoritative) artifact. Data-plane entities (vectors, campaign/5etools
data) are explicitly **out of scope** and only *referenced* by location, never owned.

## Entities

### ConfigEntity (authority: `platform.yaml`)
The single source of write authority (Principle V). Hand-edited. Top-level fields:

| Field | Type | Rules |
|---|---|---|
| `venv` | path | Required. The runtime venv location (e.g. `~/.venvs/main`). |
| `machines` | map<name, Machine> | Required. At least the `dgx` machine. |
| `data_roots` | map<name, path> | Paths to external data (referenced, not owned). |
| `services` | map<name, Service> | Managed or externally-checked services. |
| `components` | map<name, Component> | The six installable units. |
| `order.install` | list<component-name> | Required. Acyclic; every name must exist in `components`. |
| `order.startup` | list<service-name> | Required. Acyclic; every name must exist in `services`. |

### Machine
| Field | Type | Rules |
|---|---|---|
| `endpoint` | url | Required (e.g. `http://192.0.2.10:8001/v1`). |
| `default_model` | string | Optional (DGX default model). |

### Service
| Field | Type | Rules |
|---|---|---|
| `url` | url | Required. Used by both `up` gating and `status` reachability. |
| `port` | int | Optional (for local bind/lifecycle). |
| `managed` | bool | `true` = `platform` starts/stops it; `false` = external dependency, health-checked only (e.g. the DGX endpoint). |
| `health` | object | Optional probe override `{ type: tcp\|http, path: "/..." }`. Default: TCP to `port`/`url`. |
| `start` / `stop` | string | Required iff `managed: true`. Command to launch/stop (run in `venv`, cwd from the owning component). |

### Component
| Field | Type | Rules |
|---|---|---|
| `source` | enum + locator | Required. `pypi` (+ package name) or a local path / git URL. |
| `pin` | string | Required. Released version (`3.3.5`) or git ref (sha/tag). No `>=`/ranges. No editable installs. |
| `config_template` | path | Optional. The jinja2 template that renders this component's native config. |
| `config_target` | path | Required iff `config_template` set. Where the rendered file is written (the component's native config path). |

### DerivedConfig (NON-authoritative; produced by `render`)
A regenerated file written to a component's `config_target`. Carries a stamped header:
`# platform-rendered; source-sha256: <hash of the platform.yaml subtree>; do-not-edit`.
- **Never** hand-edited. Regenerated by `install`/`apply`.
- Coherence rule (Principle V): a write to `platform.yaml` is not complete until every
  DerivedConfig is regenerated and affected `managed` services restarted.
- Drift = stamped `source-sha256` ≠ current source subtree hash → `status` reports FAIL.

### Out of scope — Data-plane entities (named to bound the feature)
mempalace's ChromaDB vectors, turbovecdb's stored vectors, and the contents of
`data_roots` (5etools/campaigns). Each already owns its own truth. `platform.yaml`
records **where** they live (a path/endpoint = config), never **what** they contain.
`platform` does not move, centralize, back up, or own these.

## Validation rules (enforced by `config.py`)

1. **Schema**: required fields present; types correct; `pin` is an exact version or ref
   (reject ranges and `-e`).
2. **Referential integrity**: every name in `order.install` ∈ `components`; every name in
   `order.startup` ∈ `services`; every `managed: true` service has `start`/`stop`.
3. **Acyclicity**: `order.install` and `order.startup` define no cycle (Principle VI).
4. **Single authority**: no field permits an alternate authoritative store (no `lockfile:`
   pointer, no second config path that is *written back*). Templates may only be *read*.
5. **Path sanity**: `venv`, `config_target`, `data_roots` resolve to absolute paths after
   `~`/env expansion.

## Relationships

```
ConfigEntity (platform.yaml)  ──1:N──>  Component ──renders──>  DerivedConfig (cache)
        │                                   │
        ├──1:N──> Machine                   └── installed into ──> venv (observed by status)
        ├──1:N──> Service  ──health-check──>  status / up-gating
        └──1:N──> data_roots (referenced only; data-plane out of scope)
```

## State transitions (operational)

```
(absent) ──install──> INSTALLED ──up──> RUNNING ──down──> INSTALLED
INSTALLED|RUNNING ──edit platform.yaml + apply──> re-rendered (+ restart if RUNNING)
any ──status──> reports {installed_version vs pin, service reachability, render drift}
```
A failed `install`/`up` transition reports FAIL and the offending unit; it never advances
state on a partial/unverified result (Principle I).
