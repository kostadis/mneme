# Contract: the mneme-owned recipe

The shared best practice, **owned by mneme** and versioned in the package (FR-007/015): `mneme/recipes/mempalace.recipe.v1.yaml`. Read-only at runtime. The prose `MEMPALACE_HOWTO.md` remains its human rationale; this file is the enforceable counterpart.

## Two layers

- **mechanical** — applies to every campaign and is **enforced** (a deviation is non-conformance unless dispositioned `deliberate`).
- **scaffold** — **recommended** and **overridable**; a deviation is a normal choice, not a problem (FR-007).

## Schema

```yaml
version: "1.0.0"                      # semver; stamped into renders; compared in conformance

mechanical:
  baseline_exclusions:                # always-ignore globs (HOWTO Step 3)
    - summaries/
    - logs/
    - voice/
    - examples/
    - notes/
    - .claude/
    - MEMPALACE.md
    - "*.sh"
    - config.yaml
    - ui_config.yaml
    - entities.json
  mining_order: subscopes_before_root # the only supported value in v1 (FR-004)
  tunnel_rooms: [npcs, world]         # shared room names that create cross-wing tunnels
  hazards:                            # documented operational hazards mneme designs around
    - id: init_overwrites_root_yaml
      note: "mempalace init regenerates root mempalace.yaml; the derived file is regenerated by mneme anyway."

scaffold:
  patterns:
    - id: three_wing                  # recommended for CampaignGenerator campaigns
      wings:
        - name: narrative
          trust: authoritative
          source_hint: docs/chapters
          rooms_hint: [chapters, general]
        - name: chronicle
          trust: accelerator
          source_hint: docs/distill_extractions
          rooms_hint: [npcs, world, arcs, general]
        - name: "<campaign>"
          trust: reference
          source_hint: .
          rooms_hint: [npcs, world, arcs, mechanics, dead, general]
    - id: two_wing                    # campaigns without an extraction pipeline
      wings: [ narrative, "<campaign>" ]
    - id: single_wing                 # campaigns with no pipeline at all
      wings: [ "<campaign>" ]
```

## Divergence keys (what conformance compares)

Conformance (`conform.py`) emits stable keys that dispositions reference (FR-027). Examples:

| Key | Layer | Meaning |
|---|---|---|
| `mechanical.exclusions.<glob>.missing` | mechanical | a baseline exclusion not in effect for the campaign |
| `mechanical.order.violation` | mechanical | wing sources not in sub-scopes-before-root order |
| `mechanical.tunnel.<room>.absent` | mechanical | a tunnel room missing where it should bridge wings |
| `scaffold.pattern.<id>.partial` | scaffold | campaign partially matches a recommended pattern (advisory) |
| `scaffold.wing.<name>.absent` | scaffold | a recommended wing not present (advisory) |

- A **mechanical** divergence is a FAIL unless a `deliberate` disposition covers its key.
- A **scaffold** divergence is informational; it is a FAIL only as `undispositioned` if the campaign also lags the recipe version (i.e. "upgrade available, not yet adopted" with no recorded reason).

## Versioning

Semver. A new MINOR/MAJOR recipe is what `publish` rolls out (FR-009); `status` compares a campaign's `recipe_version` against the package's current to surface "upgrade available."
